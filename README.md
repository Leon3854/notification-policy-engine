# Движок политик уведомлений

Это полноценный боевой микросервис, с проблеми: 
> работа с часовыми поясами (таймзоны),<br/> 
> глобальные политики блокировок,<br/> 
> логирование (Observability), <br/>
> идемпотентность и полное покрытие тестами.<br/>
	
**Главный фокус — на <u>качестве архитектуры</u>, типов, бизнес‑логики... Docker приветствуется... Форматы можно менять.**

## В чем суть задания? - В таймзонах 

Надо решить проблему часовых поясов для того что бы пользователь живущий в Париже или для жителя Москвы сообщение приходло с учетом временни по его региону. Простыми словами сообщения должны приходить с учетом часового пояса, в разрешонное время.

Логика что была исользована в самом сервисе имее большой потенциал в работе ее можно будет использовать в различных сервисах где нужны будут уведомления. Прекрасно подойдет для платежного сервиса да и не только.

**В данной работе будет разработка только backend стороны без работ на клиентской стороне.**

## Запуск сервиса:

1. В корне проекта находится файл окружения переименуйте его **.youreEnv** в **.env** и пропишите свои вводные
2. ```bash
   docker-compose up -d --build
   ```
   Для чего эта команда: Для запуска всего приложения через одну команду

### Что, стоит доработать (на будущее):

Вот несколько моментов, которые будут добавлены и помогут довести сервис до идеала:

**1. Дефолтные настройки для новых пользователей**
В коде `message.service.ts` для нового пользователя задаётся жёсткий объект с константами.
Было бы не плохо вообще такой объект использовать только как одно из условий и отработать определение по геопозиции пользователя его часовой пояс сразу по умолчанию. 
*  Завести отдельную таблицу/модель `DefaultPreferences` в БД и при создании пользователя копировать их. Это сделает систему гибче — администратор сможет менять дефолты без перевыпуска кода.

**2. Идемпотентность команд изменения настроек**
Идемпотентность реализована через Redis для метода `evaluate`- это защита от дублей запроса. Надо будет проверить и улучшить. Возможно надо добавить новые условия и проверить не вызывает ли ошибки и просто возвращает текущее состояние.
